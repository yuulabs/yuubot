Interaction Pipeline

PHASE 1 Clarify + Quick-Verify
U User describes problem / gives repro steps
A Agent immediately verifies (run command, check file)
A If no repro → Agent reports gap → User clarifies
REPEAT until info is sufficient
A Agent presents understanding + exit criteria
User
Approves
PHASE 2 Agent Executes Silently
A Plan → implement → validate → handoff
— Minimize interruptions — no progress pings
⚠ Exception: Unexpected Blocker
Third-party behavior differs from docs → interrupt user.
"Expected X, got Y. Options: {A, B}. Which direction?"

Cross-Cutting: Exit Criteria & Verification Gates

Every task must answer: "How do I know this is done?" — behavioral, verifiable, falsifiable.

✅ [ ] pytest tests/test_export.py passes (12 tests)
✅ [ ] curl /api/export returns valid PDF (not JSON error)
✅ [ ] Export 10k rows in < 5s
❌ [ ] Export feature is implemented
❌ [ ] Performance is improved
Too vague — can't verify, can't falsify

Before Stage 1 Exit — Check All Gates

☐Bug: repro steps exist AND agent has reproduced the issue
☐Feature: at least ONE concrete use case is fully articulated
☐Refactoring: user confirmed understanding of current architecture
☐All tasks: exit criteria defined and agreed upon
📁
War Room: For complex discussions, agent creates warroom/clarify.md (user can edit in-place) and warroom/clarify.html (interactive diagrams).
This avoids "copy-paste from chat" friction for non-trivial alignment.

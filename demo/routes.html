Ingress Rules Routes
How routing works: Every incoming message carries a MessageSource(producer, id, path) and a kind. Rules are matched top-down using fnmatch glob patterns on source_id, source_path, and kind. Multiple rules can match — the message is delivered to all matched actors (deduped). Enabled actors also receive an implicit system:<actor_id> rule.

# Image Addon

管理表情包和图片收藏。全局共享，不区分 ctx。

## 可用命令

### 保存图片
`img save <path> --desc "描述" [--tags "猫,搞笑"]`

- path: 图片路径，支持 `file:///path` URI 或裸路径 `/path`
- desc: 图片描述，用于后续搜索
- tags: 逗号分隔的标签，可选

### 搜索图片
`img search "<query>" [--tags "猫"] [--limit 5]`

按描述关键词和/或标签搜索。返回图片路径和描述。

### 删除图片
`img delete <id>`

按 ID 删除图片。

### 列表
`img list [--tags] [--limit 20]`

- `--tags`: 显示所有标签及数量
- 不加 `--tags`: 显示最近图片

## 使用流程

### 收藏表情包
如果你发现群友发了有趣的图，或者消息记录中有有趣的图片，注意 `<image url="file:///path/to/img.jpg"/>`：
1. 直接使用 `file:///path` 格式的路径
2. `img save file:///path/to/img.jpg --desc "猫猫表示无语" --tags "猫,无语"`

### 发送表情包
1. `img search "猫"` 找到合适的图片
2. 用 im send 发送：`echo '[{"type":"image","file":"file:///path/to/img.jpg"}]' | ybot im send --ctx <ctx_id>`

### 描述规范

描述的目的是**让未来的你能搜到这张图**。写描述时，想象你一个月后想用这张图，你会怎么搜它。

**描述必须包含：**
- 画面内容：画了什么、谁在做什么
- 情绪/氛围：表达什么感情（无语、开心、惊讶、悲伤、嘲讽…）
- 适用场景：什么时候会用到这张图

**禁止只写来源：** "piova分享的图片"、"群友发的动图" 这类描述完全没有检索价值，保存后再也找不到。

**示例对比：**

| 坏描述 | 好描述 |
|--------|--------|
| piova分享的图片 | 猫咪歪头困惑表情，适合表示"这是什么情况" |
| 群友发的动图 | 角色捂脸摇头，表示无语/无奈 |
| 表情包 | 柴犬举牌写着"就这？"，嘲讽对方操作失误 |
| PN2300分享的动图 | 女孩张大嘴巴惊讶，适合表示震惊或不敢相信 |

**标签选用通用分类词：** 猫, 狗, 柴犬, 搞笑, 无语, 可爱, 惊讶, 嘲讽, 悲伤, 表情包, 动图, 二次元

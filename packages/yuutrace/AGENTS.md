# packages/yuutrace

LLM-oriented observability SDK plus a React UI. The SDK wraps OpenTelemetry
into coarse-grained context managers (`conversation` → `turn` → `entity` /
`tool_batch`) so agent runtimes can emit structured spans without juggling raw
OTEL APIs. The UI (`ui/`) is a Vite app that browses the SQLite trace DB; built
twice — a library bundle (`ui/dist/lib`) embedded into other apps and a
standalone `ytrace ui` bundle (`ui/dist/app`). CLI entrypoint: `ytrace`.

This package is a workspace member of the monorepo at the repo root. `yuubot`
emits traces through it (see `apps/yuubot/.../core/observability.py`); the
`ytrace` CLI is also surfaced as `ybot trace ui`.

## Source Map (`src/yuutrace/`)

| Path | Responsibility |
|---|---|
| `init.py` | `init(endpoint, service_name, ...)`, `disable()`, `init_memory()` → `MemoryTraceStore`, `is_enabled/is_disabled/is_initialized`, `TracingNotInitializedError`. The only place OTEL globals are configured. |
| `context.py` | `conversation(*)`/`start_conversation` → `ConversationContext`; `.turn(role)` → `TurnContext`; `.start_entity(...)` → `EntityContext`; `.tool_batch()` → `ToolsContext`; `ToolSpan`. The coarse-grained context managers all SDK consumers use. |
| `span.py` | `current_span()`, `add_event(name, attrs)`, `NoActiveSpanError`. Low-level span handle the context managers build on. |
| `otel.py` | OTEL bridge: span creation/exporter wiring, OTEL attribute key constants. **Centralise attribute keys here** — do not duplicate them across callers. |
| `cost.py` | `record_cost`, `record_cost_delta`, `record_llm_cost` — cost/price spans. |
| `usage.py` | `record_llm_usage`, `record_tool_usage` — token/tool-use deltas. |
| `types.py` | `CostCategory`, `Currency`, `LlmCost`, `LlmUsage`/`LlmUsageDelta`, `CostDelta`, `ToolUsageDelta`. Shared typed records. |
| `memory.py` | `MemoryTraceStore` — in-process trace store for tests / no-export runs. |
| `_typing.py` | Local typing helpers (avoid circular imports). |
| `cli/db.py` | SQLite trace DB schema/read queries shared by `server` + `ui`. |
| `cli/server.py` | `run_server(db_path, host, port)` — OTEL collector HTTP server (`/v1/traces`). `shutdown_server()` for clean stop. |
| `cli/ui.py` | `run_ui(db_path, host, port)` — serves the built `ui/dist/app` bundle and the trace browse API. `shutdown_ui()` for clean stop. |
| `cli/main.py` | `ytrace` CLI wiring (`ytrace server`, `ytrace ui`, ...). |
| `cli/_static/` | Packaged static assets served by the CLI (do not edit by hand — regenerated by `scripts/build_ui.sh`). |

### `ui/` — React trace browser (Vite + TypeScript)

| Path | Responsibility |
|---|---|
| `ui/src/` | React app + library source. |
| `ui/vite.config.ts` | Standalone `ytrace ui` app build → `ui/dist/app`. |
| `ui/vite.config.lib.ts` | Library build → `ui/dist/lib` (embedded into other apps). |
| `ui/package.json` | UI deps; built with npm. |

## Trace flow (quick reference)

```text
Agent runtime (yuuagents → yuubot YuubotTraceContextProvider)
  → yuutrace.conversation(id, agent, model) / .turn(role) / .start_entity / .tool_batch
    → span.py current_span() → otel.py creates OTEL spans + sets attrs
      → init.py exporter pushes to OTEL collector
        → cli/server.py (on http://host:4318/v1/traces) writes to cli/db.py SQLite
          → cli/ui.py (ytrace ui) reads the SQLite DB → React UI renders
```

Invariants to preserve:

- OTEL attribute keys live in `otel.py` (and the yuubot-layer
  `yuubot.*` attrs in `apps/yuubot/.../core/observability.py`). Do not invent
  ad-hoc keys elsewhere.
- `init()`/`disable()` is the only OTEL globals mutation site. Host apps call
  it once at startup; everything else reads via `current_span()`/`add_event`.
- The CLI static assets in `cli/_static/` are regenerated by
  `scripts/build_ui.sh` after UI changes — do not hand-edit.

## Commands

```bash
uv sync                 # from monorepo root
uv run ytrace --help
uv run ytrace server --db ./traces.db --port 4318
uv run ytrace ui --db ./traces.db --port 8080
uv run pytest
uv run pytest tests/test_<x>.py -v
uv run ruff check src/ tests/
uv run ruff format src/ tests/
```

UI (npm; the monorepo `.tmp/cache/npm` is used when built via `ybot dev`):

```bash
cd ui
npm ci
npm run typecheck
npm run build
npm run dev
bash ../scripts/build_ui.sh        # after UI changes that affect packaged assets
```

Use `examples/weather_agent.py` for a manual smoke check when trace output
changes.

## Coding style

Python: `from __future__ import annotations`, type hints on public APIs,
`msgspec.Struct` for serializable data, `snake_case`/`PascalCase`/
`UPPER_SNAKE_CASE`. UI (TypeScript): `PascalCase` components, `camelCase`
helpers, double-quoted strings. Keep commits scoped; call out schema, OTEL
attribute, or packaged-asset impact in PRs.
